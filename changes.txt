PART 4: THE "HIDDEN" IMAGES (LAZY LOADING)
-------------------------------------------
Path: class Core -> def fix_lazy_loaded_images() (Lines 1367-1375)
Path: class Product -> def images() (Modified inside logic)

1. WHAT CHANGED:
   - Updated the code to look for 'data-src' or 'data-lazy-src' attributes.
   - Added logic to skip 'data:image' (base64) strings.

2. WHY:
   - Modern sites use "Lazy Loading". The real image URL is hidden in 'data-src' 
     and only moves to 'src' when you scroll down. Python doesn't scroll, so it 
     finds the hidden URL instead.

3. HOW TO DO IT YOURSELF:
   - If you download an image and it's just a tiny 1x1 grey pixel or a blank 
     square, it means the site is lazy-loading. Check the HTML for 'data-src'.


ARCHITECTURAL EXPLANATION (FOR SENIOR DEVELOPER REVIEW)
=======================================================

When presenting this to your senior, you can explain your technical approach 
using these 4 bullet points:

1. CONFIGURATION-DRIVEN EXTRACTION (THE "MAP")
   "I used a Configuration-Driven approach. Instead of hard-coding logic for 
   every page, I built a 'SITE_CONFIG' dictionary. This decouples the website's 
   HTML structure (selectors) from the crawling engine. If the website changes its 
   UI tomorrow, we only need to update the CSS selectors in one place, not 
   modify the entire script."

2. HEURISTIC DETECTION LOGIC
   "The script implements automatic page-type detection based on the DOM structure 
   rather than the URL pattern. It checks for specific 'hero' elements (like 
   .split-products) to distinguish between Category, Group, and Product pages. 
   This makes the scraper more robust against non-standard URL structures."

3. ROBUST DATA NORMALIZATION & FALLBACKS
   "For the product listings, I implemented a 'Fallback Heuristic'. If the primary 
   CSS container (the grid) isn't found, the code automatically scans the page 
   for any links following the '/product/' pattern. This ensures we capture 
   orphaned product links that might be missing from the main grid."

4. DATA INTEGRITY & ASSET MANAGEMENT
   "The scraper doesn't just pull text; it manages the full asset lifecycle. 
   I used 'filetype' detection to verify downloads, handled 'Lazy Loading' by 
   targeting data-attributes (data-src), and used PyPDF2 to extract structured 
   metadata (version/date) directly from PDF properties for the metadata.json."
